# 初识自动化测试

说来惭愧,两年开发经验,却是 0 的测试经验. 借这次机会,好好学习下 Jest 吧.

# 常用用法

# 初始化 demo 构建

  1. 安装依赖包

    npm install jest -D

  2. 写一个被测试文件mydemo.js

    function foo(money) {
  return money > 1000 ? 'rich' : 'poor'
}

function bar(age) {
  return age > 50 ? 'old' : 'young'
}

module.exports = {
  foo,
  bar,
}

  3. 写一个测试文件mydemo.test.js

    const { foo, bar } = require('../src/myDemo1')

test('shold be rich', () => {
  expect(foo(1005)).toBe('rich')
})
test('shold be rich', () => {
  expect(foo(10)).toBe('poor')
})

  4. 运行测试

    npx jest

    或者在package.json的scripts中配置test:'jest'后运行

    npm run test

# 显示测试覆盖率

如果按照上面的配置完后,我们会发现虽然通过了测试,但是没有显示测试覆盖率报告,导致我们无法得知我们写的代码和测试案例是否做到了完美. 所以在添加一点配置来显示覆盖率,从而完善我们的测试案例.

  1. 创建jest.config.js文件

    npx jest --init

    此时会让我们配置一些选项

  2. 然后将jest.config.js中下面collectCoverage设置为true

  3. 此时再次运行npm run test会发现测试覆盖率只有 50% 因为我们下面的测试案例只有测试了 if 判断的一半,没测试另一半,所以才会出现 50%的情况

    const { foo, bar } = require('../src/myDemo1')

test('shold be rich', () => {
  expect(foo(1005)).toBe('rich')
})
test('shold be rich', () => {
  expect(foo(10)).toBe('poor')
})

    我们添加两个下面测试案例然后再运行一次npm run test

    test('shold be young', () => {
  expect(bar(51)).toBe('old')
})
test('shold be young', () => {
  expect(bar(30)).toBe('young')
})

    此时就完美的 100%啦!

  4. 每次修改完测试案例都需要重新运行npm run test有点麻烦,所以可以用下面的命令来自动运行测试

    npx jest --watchAll
//或者
npx jest --watch

# 常用匹配器

  1. toBe(xx) : 绝对相等,相当于===,字符串,数字,布尔值时使用
  2. toEqual(xx) : 对象或者数组之间的内容相等时使用
  3. toBeNull() :匹配null
  4. toBeUndefined() : 匹配undefined
  5. toBedefined() :只要定义了都能通过
  6. toBeTruthy() : 只要是 true 就能过
  7. toBeFalsy(): 只要是 false 就能过
  8. toBeGreaterThan(xx): 只要是大于 xx 就能过
  9. toBeLessThan(xx) : 只要是小于 xx 就能过
  10. toBeGreaterThanOrEqual(xx): 只要是大于等于 xx 就能过
  11. toBeLessThanOrEqual(xx): 只要是小于等于 xx 就能过
  12. toBeCloseTo(xx) : 专治 javascript 的浮点数精准问题,如:为什么 0.1+0.2 不等于 0.3?,此时就可以用此匹配器让0.1+0.2=0.3
  13. toMatch(xx): 匹配某字符串中是否有 xx
  14. toContail(xx) 匹配某数组中是否有 xx 常用!
  15. toThrow() : 如果匹配到异常则能通过,参数可以为空也可以为Error原型对象,也可以是字符串或者正则

# 钩子函数

  1. beforeAll() 在所有测试用例之前执行
  2. afterAll() 在所有测试用例之后执行
  3. beforeEach() 在每个测试用例之前执行
  4. afterEach() 在每个测试用例之后执行

# Mock 函数

  1. 可以用来测试函数的参数与返回值

    const mockCallback = jest.fn(x => 42 + x)
forEach([0, 1], mockCallback)

// The mock function is called twice
expect(mockCallback.mock.calls.length).toBe(2)

// The first argument of the first call to the function was 0
expect(mockCallback.mock.calls[0][0]).toBe(0)

// The first argument of the second call to the function was 1
expect(mockCallback.mock.calls[1][0]).toBe(1)

// The return value of the first call to the function was 42
expect(mockCallback.mock.results[0].value).toBe(42)

  2. 还可以给自定义测试返回值

    const myMock = jest.fn()
console.log(myMock())
// > undefined

myMock
  .mockReturnValueOnce(10)
  .mockReturnValueOnce('x')
  .mockReturnValue(true)

console.log(myMock(), myMock(), myMock(), myMock())
// > 10, 'x', true, true

# Mock 模块

// users.js
import axios from 'axios'

class Users {
  static all() {
    return axios.get('/users.json').then(resp => resp.data)
  }
}

export default Users

// users.test.js
import axios from 'axios'
import Users from './users'

jest.mock('axios')

test('should fetch users', () => {
  const users = [{ name: 'Bob' }]
  const resp = { data: users }
  axios.get.mockResolvedValue(resp)

  // or you could use the following depending on your use case:
  // axios.get.mockImplementation(() => Promise.resolve(resp))

  return Users.all().then(data => expect(data).toEqual(users))
})
